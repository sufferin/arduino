# Управление семисегментным индикатором через каскад 74HC595

## Описание решения

В данном проекте реализован двузначный секундомер (00-99), управляемый микроконтроллером ATmega328P (Arduino Uno).
Ключевой особенностью является **отсутствие блокирующих функций** (`delay`) и стандартных Arduino-методов (
`digitalWrite`, `shiftOut`). Все управление временем, сдвигом битов и обновлением дисплея выполняется на низком уровне
через прямую запись в регистры портов и прерывания таймера.

### Аппаратная архитектура

Используются два сдвиговых регистра 74HC595 в каскадном включении:

1. **Arduino Pin 8 (DS)** подключен ко входу данных первого регистра.
2. **Выход Q7'** первого регистра подключен ко входу данных (DS) второго регистра.
3. **Clock (SH_CP)** и **Latch (ST_CP)** обоих регистров соединены параллельно и управляются общими линиями.

Это позволяет передавать 16 бит данных по 3 проводам: первые 8 бит "проталкиваются" во второй регистр (отвечает за
десятки), следующие 8 бит остаются в первом (отвечает за единицы).

### Программная логика

Система построена на архитектуре **Producer-Consumer**:

1. **Главный цикл (`loop`) — Producer (Поставщик данных):**
    * Слушает Serial порт.
    * Парсит входящие символы.
    * При получении валидного числа (00-99) атомарно обновляет переменную `next_seconds`.
    * Не занимается отрисовкой или отсчетом времени.

2. **Таймер Timer1 (ISR) — Consumer (Потребитель и Исполнитель):**
    * Настроен в режим CTC (Clear Timer on Compare Match) с частотой 1 кГц (1 мс).
    * **Счетчик времени:** Каждые 1000 тиков (1 секунда) берет значение из `next_seconds`, делает его текущим (
      `current_seconds`) и инкрементирует `next_seconds` для следующего цикла.
    * **Машина состояний (State Machine):**
        * Когда наступает новая секунда, данные (цифры) преобразуются в битовую маску (16 бит) согласно карте сегментов.
        * Запускается процесс `SHIFTING`. В каждом прерывании (каждую 1 мс) выставляется 1 бит данных и дергается ножка
          Clock.
        * После передачи 16 бит состояние переходит в `LATCHING`, где подается импульс Latch для фиксации данных на
          выходах.

   *Почему так?* Распределение сдвига битов по времени (1 бит за 1 мс) делает процесс обновления абсолютно прозрачным
   для основного процессора и гарантирует равномерную загрузку CPU, соответствуя требованию о "процессах, управляемых
   таймерами".

### Схема подключения

| Функция 74HC595    | Пин 74HC595 | Arduino Pin    | Порт AVR |
|:-------------------|:------------|:---------------|:---------|
| **DS (Data)**      | 14 (Reg 1)  | D8             | PB0      |
| **SH_CP (Clock)**  | 11 (Both)   | D9             | PB1      |
| **ST_CP (Latch)**  | 12 (Both)   | D10            | PB2      |
| **Q7' (Overflow)** | 9 (Reg 1)   | Pin 14 (Reg 2) | -        |
| **VCC / MR**       | 16, 10      | 5V             | -        |
| **GND / OE**       | 8, 13       | GND            | -        |

* **Резисторы:** 220 Ом на каждый сегмент (между выходами Q0-Q7 и ножками индикатора).
* **Индикаторы:** Общий анод или катод (выберите нужное значение `COMMON_ANODE` в коде).

### Инструкция по запуску

1. Соберите схему согласно таблице.
2. Проверьте константу `COMMON_ANODE`:
    * `true` — если общий провод индикатора подключен к +5V.
    * `false` — если общий провод подключен к GND.
3. Загрузите прошивку.
4. Откройте Монитор порта (9600 бод).
5. Индикатор начнет счет с 00. Введите любое число (например, `45`) и нажмите Enter. При следующей смене секунды счетчик
   перепрыгнет на 45.
