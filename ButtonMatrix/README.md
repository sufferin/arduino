# Реализация драйвера матричной клавиатуры (3x3) на прерываниях и портах

## 1. Введение и постановка задачи

### 1.1. Цель проекта
Целью данного проекта является разработка высокоэффективного драйвера для матричной клавиатуры (размерностью 3x3 или больше) на микроконтроллере ATmega328P (Arduino Uno). Ключевое требование — отказ от стандартных блокирующих методов опроса и медленных функций Arduino API в пользу прямой работы с регистрами портов и аппаратными таймерами.

Задача заключается в создании системы, которая сканирует состояние кнопок в фоновом режиме, поддерживает одновременное нажатие нескольких клавиш (multitouch) и предоставляет точную информацию о времени нажатия и удержания каждой кнопки, не занимая ресурсы основного цикла `loop()`.

### 1.2. Проблема стандартных подходов
Классические методы опроса матриц (например, вложенные циклы `for` с использованием `digitalRead` и `delay`) имеют существенные недостатки:
*   **Блокировка процессора:** Использование `delay()` для устранения дребезга контактов останавливает выполнение других задач.
*   **Низкая производительность:** Функция `digitalRead` потребляет около 50 тактов процессора, что при частом опросе 9 кнопок создает значительную нагрузку.
*   **Сложность масштабирования:** В простых реализациях трудно отслеживать длительность нажатия для каждой кнопки индивидуально при одновременном нажатии.

### 1.3. Предложенное решение
Для реализации неблокирующего опроса используется механизм аппаратных прерываний **Таймера1**. Таймер настроен на регулярное срабатывание (каждые 5 мс), вызывая обработчик прерывания (ISR). Внутри ISR происходит переключение активной строки и мгновенное считывание состояния столбцов через регистры ввода-вывода.
Основной цикл программы `loop()` освобожден от работы с "железом" и занимается только высокоуровневой логикой: детектированием фронтов сигнала (нажатие/отпускание) и логированием событий.

## 2. Архитектура и программная реализация

### 2.1. Управление таймером (Timer1)
16-битный **Таймер1** работает в режиме CTC (Clear Timer on Compare Match) и является тактовым генератором для сканера матрицы.
*   **Конфигурация:** Предделитель 64, регистр сравнения `OCR1A = 1249`.
*   **Интервал сканирования:** При частоте 16 МГц это обеспечивает вызов прерывания каждые `5 миллисекунд`.
*   **Принцип действия:** Каждое прерывание отвечает за опрос ровно одной строки матрицы. Полный цикл опроса всей матрицы 3x3 занимает 15 мс, что достаточно быстро для реакции человека и естественно устраняет эффект "дребезга" контактов.

### 2.2. Прямое управление портами (Direct Port Manipulation)
Для максимальной скорости вместо `digitalWrite`/`digitalRead` используются регистры порта D:
*   **Строки (Outputs):** Пины D2, D3, D4. Управляются через регистр `PORTD`. Логика работы — "Бегущий ноль" (Active Low).
*   **Столбцы (Inputs):** Пины D5, D6, D7. Считываются через регистр `PIND`. Включены внутренние подтягивающие резисторы (Input Pull-up).
*   **Битовые операции:** Состояние всей строки считывается одной командой `uint8_t pin_state = PIND`, что занимает всего 1-2 такта процессора.

### 2.3. Логика хранения состояния (Битовая карта)
Состояние всех 9 кнопок хранится в одной переменной `volatile uint16_t button_state_bitmap`.
*   Каждый бит соответствует одной кнопке (индексы 0–8).
*   ISR обновляет эту переменную атомарно.
*   Использование `uint16_t` необходимо, так как кнопок 9, и они не помещаются в стандартный байт (8 бит).

### 2.4. Обработка событий (Edge Detection)
В основном цикле реализован детектор фронтов сигнала для выполнения требований ТЗ (пункты 1 и 2):
1.  **Rising Edge (Нажатие):** Текущий бит 1, старый 0. Фиксируется системное время `millis()` в массив `btn_start_times`.
2.  **Falling Edge (Отпускание):** Текущий бит 0, старый 1. Вычисляется разница `current_time - start_time`, результат выводится в Serial Monitor.
3.  **State Change:** При любом изменении состояния выводится полный список удерживаемых кнопок.

## 3. Тестирование и верификация

### 3.1. Методология тестирования
Тестирование проводилось в симуляторе Autodesk Tinkercad.
*   **Схема:** Arduino Uno, 3 ползунковых переключателя (для симуляции строк/столбцов и одновременных нажатий), диоды 1N4148 для защиты от замыканий (Anti-ghosting).
*   **Инструментарий:** Serial Monitor для отслеживания логов.

### 3.2. Сценарии проверки
Были успешно проверены следующие сценарии:
1.  **Одиночное нажатие:** Кнопка нажимается и отпускается. Система выводит событие нажатия, затем событие отпускания с точной длительностью в миллисекундах.
2.  **Мультитач (Аккорды):** Нажатие кнопки 1, затем добавление кнопки 5, затем кнопки 9. Система корректно обновляет список: `[1]`, `[1, 5]`, `[1, 5, 9]`.
3.  **Асинхронное отпускание:** При зажатых 1, 5, 9 отпускается кнопка 5. Система выводит длительность только для кнопки 5 и обновляет список оставшихся: `[1, 9]`.
4.  **Проверка 9-й кнопки:** Подтверждена работа кнопки с индексом 8 (9-я по счету), что доказывает корректность использования 16-битной переменной состояния.

### 3.3. Результаты
Система демонстрирует стабильную работу без пропуска нажатий. Вывод в Serial Monitor происходит строго по событиям, не засоряя эфир повторяющимися сообщениями, что соответствует требованиям "Point 1" и "Point 2".

## 4. Исходные коды и ресурсы

### 4.1. Полный листинг программы (Main.ino)

```cpp
/*
 * IoT Матричная Клавиатура - Реализация (Пункты 0, 1, 2)
 * Функционал: Отслеживание длительности, времени начала, мультитач.
 * Архитектура: Сканирование в прерываниях Timer1 + Прямой доступ к PORTD.
 */

#include <avr/io.h>
#include <avr/interrupt.h>

// --- Конфигурация ---
#define NROWS 3
#define NCOLS 3

// --- Глобальное состояние ---
// Хранит "живое" состояние кнопок (обновляется мгновенно в ISR)
volatile uint16_t button_state_bitmap = 0; 
volatile int current_row = 0;

// Массив для хранения времени начала нажатия
unsigned long btn_start_times[9];

// Макрос для перевода координат в индекс
#define GET_BTN_INDEX(row, col) ((row * NCOLS) + col)

void setup() {
  Serial.begin(9600);

  // --- 1. Настройка Портов ---
  // Строки (Выходы): D2, D3, D4 | Столбцы (Входы): D5, D6, D7
  DDRD |= 0x1C;  // D2-D4 OUTPUT
  DDRD &= ~0xE0; // D5-D7 INPUT
  PORTD |= 0x1C; // Строки HIGH (неактивны)
  PORTD |= 0xE0; // Включаем Pull-ups на входах

  // --- 2. Настройка Timer1 (CTC Mode, 5ms) ---
  noInterrupts();
  TCCR1A = 0; TCCR1B = 0; TCNT1  = 0;
  OCR1A = 1249;             // (16MHz / (64 * 200Hz)) - 1
  TCCR1B |= (1 << WGM12);   // CTC Mode
  TCCR1B |= (1 << CS11) | (1 << CS10); // Prescaler 64
  TIMSK1 |= (1 << OCIE1A);  // Enable Interrupt
  interrupts();
  
  Serial.println("Система готова. Драйвер клавиатуры запущен.");
}

void loop() {
  static uint16_t last_processed_state = 0;
  uint16_t current_state_copy;

  // Атомарное чтение
  noInterrupts();
  current_state_copy = button_state_bitmap;
  interrupts();

  // Логика обработки изменений
  if (current_state_copy != last_processed_state) {
    unsigned long current_time = millis();

    for (int i = 0; i < 9; i++) {
      bool is_pressed_now = (current_state_copy >> i) & 1;
      bool was_pressed_before = (last_processed_state >> i) & 1;

      // Нажатие (Rising Edge)
      if (is_pressed_now && !was_pressed_before) {
        btn_start_times[i] = current_time;
      }

      // Отпускание (Falling Edge)
      if (!is_pressed_now && was_pressed_before) {
        unsigned long duration = current_time - btn_start_times[i];
        Serial.print(">> Кнопка "); Serial.print(i + 1);
        Serial.print(" отпущена. Длительность: "); Serial.print(duration);
        Serial.print(" мс. Начало в: "); Serial.print(btn_start_times[i]);
        Serial.println(" мс.");
      }
    }

    printCurrentList(current_state_copy);
    last_processed_state = current_state_copy;
  }
  delay(50);
}

// Помощник вывода списка
void printCurrentList(uint16_t state) {
  if (state == 0) { Serial.println("Статус: Все кнопки отпущены."); return; }
  Serial.print("Статус: нажаты кнопки [ ");
  bool first = true;
  for (int i = 0; i < 9; i++) {
    if ((state >> i) & 1) {
      if (!first) Serial.print(", ");
      Serial.print(i + 1); first = false;
    }
  }
  Serial.println(" ]");
}

// ISR Сканирования
ISR(TIMER1_COMPA_vect) {
  PORTD |= (1 << (current_row + 2)); // Выкл текущую строку
  current_row++;
  if (current_row >= NROWS) current_row = 0;
  PORTD &= ~(1 << (current_row + 2)); // Вкл новую строку

  uint8_t pin_state = PIND; // Чтение всех входов
  for (int col = 0; col < NCOLS; col++) {
    bool is_pressed = !(pin_state & (1 << (5 + col)));
    int btn_index = GET_BTN_INDEX(current_row, col);
    
    if (is_pressed) button_state_bitmap |= (1U << btn_index); 
    else button_state_bitmap &= ~(1U << btn_index); 
  }
}
```
