## 1. Введение
При проектировании автономных робототехнических систем одной из главных проблем является вариативность аппаратного обеспечения. DC-моторы одного типа могут иметь разную скорость вращения, а полярность их подключения часто определяется только опытным путем. 

В данном проекте реализована **профессиональная система калибровки**, которая позволяет настроить:
1. **Направления вращения** (логика DIR-пинов).
2. **Баланс скоростей** (компенсация увода платформы в сторону).
3. **Тайминги поворотов** (точное выполнение маневров на 90/180 градусов).

Все настройки сохраняются в энергонезависимую память **EEPROM**, что позволяет платформе «помнить» калибровку даже после отключения питания.

## 2. Аппаратная реализация
Система построена на базе микроконтроллера **Arduino Uno** и драйвера моторов **L293D**.
*   **Левый мотор:** Пины 4 (DIR) и 5 (PWM).
*   **Правый мотор:** Пины 7 (DIR) и 6 (PWM).
*   **Связь:** Модуль Bluetooth (HC-05/06) через `SoftwareSerial` (пины 2, 3).
*   **Питание:** Раздельное (5V для логики, 9V для силовой части моторов).

## 3. Программная архитектура (Конечный автомат)
Прошивка управляется конечным автоматом с четырьмя состояниями. Переключение между ними осуществляется кнопкой **X (символ '3')**.

### Режим 0: DRIVE MODE (Штатная эксплуатация)
Режим обычной езды. Система использует сохраненные в EEPROM индексы направлений и коэффициенты смещения скорости (Offset), обеспечивая прямолинейное движение.

### Режим 1: CALIB_DIR (Калибровка направлений)
*   **Цель:** Сопоставить команду (например, Forward) с правильной комбинацией сигналов на пинах.
*   **Логика:** Кнопкой **Y ('1')** пользователь перебирает одну из 4-х комбинаций (00, 10, 01, 11). Как только моторы начинают вращаться в нужную сторону, настройка фиксируется кнопкой **A ('2')**.

### Режим 2: CALIB_BALANCE (Балансировка скорости)
*   **Цель:** Выровнять скорость левого и правого колеса.
*   **Логика:** В движении кнопками **<** и **>** пользователь меняет значение `leftOffset`. Это позволяет программно замедлить или ускорить одно колесо относительно другого, добиваясь идеально ровного курса. Настройка сохраняется кнопкой **A ('2')**.

### Режим 3: CALIB_TURNS (Тайминги маневров)
*   **Цель:** Откалибровать время поворота на фиксированный угол (90°).
*   **Логика:** Кнопка **L** запускает тестовый поворот. Если угол меньше 90°, кнопками **Y** и **B** (вперед/назад) пользователь увеличивает или уменьшает длительность задержки (`delay`). После настройки маневр становится предсказуемым и точным.

## 4. Карта управления (Gamepad Mapping)

| Кнопка | Символ | Drive Mode | Calibration Mode |
| :--- | :--- | :--- | :--- |
| **^** | `F` | Вперед | Тест движения вперед |
| **v** | `B` | Назад | Уменьшить время поворота (в Mode 3) |
| **<** | `L` | Влево | Тест поворота / Уменьшить баланс |
| **>** | `R` | Вправо | Тест поворота / Увеличить баланс |
| **X** | `3` | **Смена режима** | **Смена режима** |
| **Y** | `1` | - | Смена комбинации DIR / Увеличить время |
| **A** | `2` | - | **СОХРАНИТЬ В EEPROM** |
| **B** | `4` | Стоп | Стоп / Отмена |

## 5. Результаты
В ходе работы была создана универсальная прошивка, исключающая необходимость перепрограммирования при смене моторов или переподключении проводов. Использование EEPROM делает систему автономной и готовой к работе сразу после включения.
